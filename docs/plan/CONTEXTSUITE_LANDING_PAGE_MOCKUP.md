# ContextSuite Landing Page Mockup Guide

## 1. Purpose
This document defines a mockup-only concept for the ContextSuite website experience. It covers layout, visual direction, messaging, and UI sections, including login and signup screens.

Scope note: this is a design and content guide only. No backend behavior, API integration, or production functionality is required.

## 2. Design Goals
- Communicate trust, reliability, and technical credibility in under 5 seconds.
- Explain ContextSuite value quickly: prevent regressions before code execution.
- Make the call-to-action clear for both individuals and teams.
- Keep the visual style modern, minimal, and product-focused.

## 3. Brand Direction
### Tone
- Confident, technical, clear.
- Avoid hype language.
- Emphasize safety, traceability, and team memory.

### Visual Style
- Clean light theme with high contrast text.
- Strong typography hierarchy.
- Product screenshot-first storytelling.
- Subtle motion (fade/slide/stagger), not flashy animations.

### Color Palette (example)
- Primary: Deep Blue (#1A4DFF)
- Accent: Mint (#18C29C)
- Warning: Amber (#F5A524)
- Surface: Off White (#F8FAFC)
- Text: Near Black (#0F172A)

## 4. Information Architecture
### Top Navigation
- Logo: ContextSuite
- Links: Product, How It Works, Pricing, Security, Docs
- Right actions: Log In, Get Started

### Page Flow
1. Hero (value proposition + CTA)
2. Problem and outcome strip
3. How ContextSuite works (3-step visual)
4. Feature grid
5. Trust and governance section
6. Pricing snapshot
7. Integrations strip
8. FAQ
9. Final CTA
10. Footer

## 5. Section-by-Section Mockup Content
### Hero Section
- Headline: "Stop AI Coding Regressions Before They Ship"
- Subheadline: "ContextSuite reviews every plan against historical issue memory, approved constraints, and risk policies."
- Primary CTA: "Get Started"
- Secondary CTA: "See Demo"
- Visual: dashboard mockup showing risk flags, approval state, and issue-linked context.

### Problem and Outcome Strip
- Left: "Teams keep re-breaking old logic"
- Right: "ContextSuite catches risky plans before execution"
- Metrics placeholders:
  - "Fewer repeated incidents"
  - "Faster onboarding"
  - "Explainable approval history"

### How It Works (3 Steps)
1. Prompt goes to Context Agent first.
2. Plan is reviewed against prior issues and constraints.
3. Approved plans execute; issue-related outcomes are indexed.

### Feature Grid
- Context-first routing
- Risk-aware plan review
- Human-in-the-loop approvals
- Git diff-aware checks
- Issue-only indexing memory
- Explainable trust scoring

### Trust and Governance Section
- Messaging: "Every decision is traceable"
- Visual cards:
  - "Who approved"
  - "Why approved"
  - "What changed"
  - "What constraints were checked"

### Pricing Snapshot
- Individual: $15/month, up to 5 repos/projects.
- Starter: $10 per seat.
- Growth: discounted per-seat tiers.
- Add-ons: Slack, Telegram, and integration plugins.
- CTA: "Talk to Sales" and "Start Free Setup"

### Integrations Strip
- Logos/placeholders: GitHub, GitLab, VS Code, Slack, Telegram, Jira
- Supporting line: "Works with your current dev workflow"

### FAQ
- "Is this another coding assistant?"
- "How does approval work?"
- "What gets indexed?"
- "Can we use it for existing repos?"

### Final CTA
- Headline: "Make AI coding safer across your team"
- Buttons: "Get Started" and "Book a Live Demo"

## 6. Login Mockup
### Goal
Quick access for existing users with clear account context.

### Layout
- Split panel layout:
  - Left panel: brand message + product visual.
  - Right panel: login form.

### Form Elements
- Email
- Password
- Remember me checkbox
- Log In button
- "Forgot password?"
- Divider: "or continue with"
- SSO buttons: Google, GitHub, Microsoft

### Helper Copy
- "Secure sign-in for ContextSuite workspaces"
- "By continuing, you agree to Terms and Privacy"

## 7. Signup Mockup
### Goal
Simple onboarding for individuals and teams.

### Step-Based Mockup (3 screens)
1. Account creation
- Name
- Work email
- Password
- Confirm password
- Continue button

2. Workspace setup
- Workspace name
- Team size selector
- Primary use case selector (bugfix, governance, compliance, onboarding)
- Continue button

3. Plan selection
- Individual / Starter / Growth / Enterprise contact
- Optional plugin checkboxes (Slack, Telegram, Jira)
- Create workspace button

### Success Screen Mockup
- "Workspace created"
- Placeholder connection endpoint panel
- "Go to Dashboard" button
